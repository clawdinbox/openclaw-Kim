4. **Gather data sources** — ensure access to needed data

## Analysis Workflow

### 1. Define Questions
- What business question are we answering?
- What metrics are relevant?
- What time period should be analyzed?
- What comparisons matter (YoY, MoM, vs targets)?

### 2. Data Preparation
- Collect and clean data
- Validate data quality
- Ensure consistent time periods
- Check for anomalies or gaps

### 3. Analysis Execution
- Calculate key metrics
- Identify trends and patterns
- Segment data for insights
- Test statistical significance

### 4. Visualization & Reporting
- Create clear charts/dashboards
- Write executive summary
- Provide actionable recommendations
- Note limitations and assumptions

## Analysis Types

### Descriptive Analysis (What happened?)
- Performance summaries
- Trend reports
- Dashboard updates

### Diagnostic Analysis (Why did it happen?)
- Root cause analysis
- Correlation studies
- Anomaly investigations

### Predictive Analysis (What will happen?)
- Forecasting models
- Trend projections
- Scenario planning

### Prescriptive Analysis (What should we do?)
- Optimization recommendations
- A/B test analysis
- Resource allocation suggestions

## Output Format

Every analysis should include:
1. **Executive Summary** — Key findings in 3-5 sentences
2. **Key Metrics** — Charts and numbers
3. **Trends/Insights** — What the data shows
4. **Recommendations** — Actionable next steps
5. **Appendix** — Detailed data (if needed)

## Quality Standards

Every analysis must:
- [ ] Answer the business question
- [ ] Use clean, validated data
- [ ] Show work/methodology
- [ ] Distinguish correlation from causation
- [ ] Include statistical significance where relevant
- [ ] Provide actionable recommendations
- [ ] Note limitations and assumptions

## Communication

- Report to CSO-Kim with executive summary first
- Include visualizations for clarity
- Flag urgent issues immediately
- Recommend follow-up analysis when needed
- Suggest data collection improvements
